仪表盘

登录后，每个角色都会看到为其定制的仪表盘。

客户经理 / 管理员 / 超级管理员仪表盘

  • 统计卡片：项目总数、进行中项目数、客户总数、合同总金额
  • 收入管道：已开票、已收款、未结清金额，并附带待审核报价和审批的提示标签
  • 客户项目组合：表格展示每个客户的项目数量、进行中项目数和合同金额
  • 需处理的发票：需要审核的发票列表，附带状态标签
  • 待处理供应商报价：等待审批的报价，附带批准/拒绝操作
  • 按状态划分的项目：按当前状态划分的项目分布

项目经理仪表盘

  • 统计卡片：我的项目、进行中项目、本周到期任务、待处理工时表
  • 待处理供应商项目申请：供应商提交的项目申请，附带批准/拒绝按钮
  • 我的项目：已分配项目列表，附带健康度指标（正常、有风险、已延迟）
  • 任务分布：按状态划分的任务分布（待办、进行中、审核中、已完成）
  • 待处理报价：等待审核的供应商报价

供应商仪表盘

  • 供应商信息卡：供应商名称、联系方式、项目数量
  • 指标卡片：合同总金额、已开票、已付款、未结清金额
  • 进行中项目：当前项目列表，附带状态和健康度指标
  • 报价：管理供应商报价（创建、提交、删除），并跟踪状态（待处理、已提交、已批准、已拒绝）
  • 最近发票：最新的供应商发票及其状态

客户仪表盘

  • 账单摘要：合同金额、已开票、已付款、未结清金额
  • 我的项目：项目卡片网格，显示状态、审批状态、任务进度条、PM/AM 负责人和截止日期
  • 最近发票：最新发票列表及状态标签
  • 项目时间线：进行中项目里程碑总览

团队成员 / 通用仪表盘

  • 统计卡片：进行中项目、未完成任务、待处理工时表、本周工时、未结清发票
  • 最近项目：快速跳转到项目及其任务数量
  • 最近发票：最近发票列表及状态

供应商项目申请

客户提案流程

对于已分配客户的项目，在项目正式激活前，必须先向客户提出范围与计费条款的提案并获得批准。这与上文的"供应商项目申请"不同——它管理的是客户对成本的确认，而非供应商的接单申请。

  1. 制定提案（PM、AM、管理员、超级管理员）— 设置项目的计费方式、预估成本、预估工时，若为按工时计费或混合计费，还需设置时薪。为按里程碑或混合计费的项目添加里程碑，并设置合同金额。
  2. 发送提案 — 在项目详情页点击发送提案。客户会收到邮件通知。如果必须设置时薪但尚未设置，发送将被阻止并提示错误。
  3. 客户审核 — 客户打开项目后会看到提案面板，可以批准、拒绝（需填写理由）或要求修订（附带备注）。
  4. 批准后 — 项目自动进入 ACTIVE（进行中）状态，计费方式、预估成本、预估工时、时薪，以及每个里程碑的合同金额、名称和截止日期均被锁定，无法再编辑。
  5. 发起新修订 — PM/AM/管理员可点击发起新修订解锁这些字段并将提案版本号加一，随后编辑并重新发送以待批准。

提案状态：DRAFT（草稿）→ SENT（已发送）→ APPROVED（已批准）/ DECLINED（已拒绝）/ REVISION_REQUESTED（要求修订）。从后三种状态中的任意一种发起新修订，提案都会回到 DRAFT 状态，且版本号递增。

客户只能查看和处理属于自己组织的项目提案。

提案历史：每次发送提案时，系统都会永久记录当时的条款与里程碑快照，以及客户的响应和任何评论。点击版本标签旁的查看历史可打开时间线，查看每一个历史版本——发送了什么内容、谁发送的、客户如何响应，以及任何拒绝理由或修订备注。任何后续修订都不会删除或覆盖历史记录；即使提案已被批准并再次修订，所有早期版本依然可见。内部员工和客户均可查看完整历史。

变更请求

项目提案批准后，客户可能会要求在原有范围之外增加额外工作（例如额外的功能或交付物）。为避免重新打开并再次锁定整个提案，系统使用独立的变更请求来处理这类需求：

  1. 发起变更请求（PM、AM、管理员、超级管理员）— 在项目详情页点击+ 新建变更请求，填写标题、可选的描述、追加成本，并可选择添加一个或多个新里程碑（名称、截止日期、金额）。
  2. 客户审核 — 客户会在项目页面看到该变更请求及其他历史请求，可以批准、拒绝（需填写理由）或要求修订（附带备注）——流程与提案审核相同。
  3. 批准后 — 变更请求中列出的所有里程碑会自动创建到项目中，项目的预估成本增加相应的追加成本。原已批准的提案及其历史记录不受影响。
  4. 被拒绝或要求修订后 — 该变更请求会永久保留，并显示客户的响应和备注。若需处理反馈，需发起新的变更请求；原请求不会被编辑或删除。

一个项目在其生命周期内可以拥有任意数量的变更请求，每一条的完整历史——包括被拒绝或被取代的记录——都会一直保留可见。

项目详情页

点击任意项目名称打开详情页，该页面包含四个标签页：

  1. 工作 — 管理里程碑和任务。创建/编辑/删除里程碑仅限超级管理员、管理员、AM 和 PM。每个里程碑可设置合同金额，用于在标记完成后向客户计费（见上文"客户提案流程"）；一旦项目提案获批，该里程碑的名称、截止日期和合同金额均被锁定。创建/编辑/删除任务（以及点击状态标签快速切换任务状态）对管理者在任意项目中均可用，团队成员、承包商和供应商联系人仅限于其被分配的项目。此标签页还可查看团队成员面板。
  2. 报价 — 该项目的供应商报价（查看、创建和管理；状态包括：PENDING 待处理、SUBMITTED 已提交、APPROVED 已批准、REJECTED 已拒绝）
  3. 预算 — 项目预算跟踪；创建和管理预算明细项
  4. 文档 — 上传和管理项目文档

任务

查看任务

任务可以以表格视图查看（带状态筛选标签：全部、待办、进行中、审核中、已完成），也可以以看板视图查看（看板风格的列布局）。使用项目下拉菜单按项目筛选。该视图会显示所有项目中的任务，无论是否分配给您——浏览不受限制，但编辑权限受限（见下文）。

快速切换状态

点击任意任务上的状态标签，可依次循环切换：TODO（待办）→ IN_PROGRESS（进行中）→ REVIEW（审核中）→ DONE（已完成）（表格视图），或在看板视图中拖拽卡片到不同列。仅当您有权管理该任务时才可用——超出您分配项目范围的任务无法编辑、删除或更改状态。

工时表

通过侧边栏的工时表进入（仅限内部员工和供应商用户）。

我的工时表

创建工时表，将您的工时记录打包提交以待审批。

  1. 点击+ 新建工时表
  2. 选择周期开始和周期结束日期
  3. 系统会自动纳入匹配的工时记录
  4. 提交工时表以待审批，或保存为草稿

工时表状态

  • DRAFT（草稿） — 尚未提交；可编辑或删除
  • SUBMITTED（已提交） — 等待审核；记录已锁定
  • APPROVED（已批准） — 已由审核人批准
  • REJECTED（已拒绝） — 已附理由被拒；请查看反馈后重新提交

点击任意工时表可展开查看其中所有关联的工时记录及工时数。

审核工时表（PM、AM、管理员、超级管理员）

待审核区域显示所有团队成员提交的工时表。

  1. 点击某个工时表以展开并审核其工时记录
  2. 点击批准以批准该工时表
  3. 点击拒绝并填写理由以退回

已批准工时表区域显示之前已批准的工时表，包括总工时、计费工时以及审批人。

发票

通过侧边栏的发票进入（所有人可见）。

标签页

  • 客户发票 — 发送给客户的发票（内部员工和客户可见）
  • 供应商发票 — 来自供应商的发票（内部员工和供应商可见）

使用状态筛选器按当前状态筛选发票。

客户发票

创建客户发票（PM、AM、管理员）

  1. 点击+ 新建客户发票
  2. 选择项目和客户
  3. 添加明细项，填写描述、数量、单价
  4. 设置税率和币种（自动从客户设置中带入）
  5. 保存为草稿

从里程碑生成（PM、AM、管理员）

对于按里程碑或混合计费的项目，点击⚡ 从里程碑生成并选择项目，系统会自动创建一张草稿发票，为每个已完成的里程碑生成一条明细项，并使用该里程碑的合同金额。已在已提交、已批准或已付款发票中向客户计费过的里程碑不会重复计入。

发票生命周期

DRAFT（草稿）──► SENT（已发送）──► APPROVED（已批准）──► PAID（已付款）
                                              └──► OVERDUE（逾期，若超过到期日）
  • DRAFT（草稿） — 可编辑和删除
  • SENT（已发送） — 已发送给客户以待审核
  • APPROVED（已批准） — 客户或 AM 已批准该发票
  • PAID（已付款） — 已收到付款
  • OVERDUE（逾期） — 已超过到期日仍未付款

供应商发票

创建供应商发票（供应商联系人）

  1. 点击+ 创建发票
  2. 从已批准的报价中选择（自动带入详细信息）
  3. 添加明细项
  4. 保存为草稿或提交

供应商发票生命周期

DRAFT（草稿）──► SUBMITTED（已提交）──► APPROVED（已批准）──► PAID（已付款）
                              └──► REJECTED（已拒绝）
                              └──► REVISION_REQUESTED（要求修订）──►（创建新版本）
  • SUBMITTED（已提交） — 等待 PM/AM 审核
  • APPROVED（已批准） — PM/AM 已批准该发票
  • REJECTED（已拒绝） — AM 已拒绝该发票（附带备注）
  • REVISION_REQUESTED（要求修订） — PM/AM 要求修改；系统会创建一个新版本供供应商编辑并重新提交

当有待处理的供应商发票时，审核队列提示条会向 PM/AM 用户提示数量和总金额。

发票详情页

点击任意发票可查看完整详情：包含状态、日期和金额的发票头信息；明细项表格；付款历史；以及根据您的角色和发票状态显示的操作按钮（提交、批准、拒绝、要求修订）。

记录付款

对于已批准的发票，授权用户可以记录付款：

  1. 打开发票详情页
  2. 点击记录付款
  3. 输入付款金额和日期
  4. 全额付款后，发票状态自动更新为 PAID（已付款）

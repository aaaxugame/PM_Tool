快速入門

登入

儀表板

登入後，每個角色都會看到為其量身打造的儀表板。

客戶經理 / 管理員 / 超級管理員儀表板

  • 統計卡片：專案總數、進行中專案數、客戶總數、合約總金額
  • 營收管道：已開立、已收款、未結清金額，並附帶待審核報價與核准的提示標籤
  • 客戶專案組合：表格顯示每個客戶的專案數量、進行中專案數與合約金額
  • 需處理的發票：需要審核的發票清單，附帶狀態標籤
  • 待處理廠商報價：等待核准的報價，附帶核准/拒絕操作
  • 依狀態劃分的專案：依目前狀態劃分的專案分布

專案經理儀表板

  • 統計卡片：我的專案、進行中、本週到期任務、待處理工時表
  • 待處理廠商專案申請：廠商提交的專案申請，附帶核准/拒絕按鈕
  • 我的專案：已指派專案清單，附帶健康度指標（正常、有風險、已延遲）
  • 任務分布：依狀態劃分的任務分布（待辦、進行中、審核中、已完成）
  • 待處理報價：等待審核的廠商報價

廠商儀表板

  • 廠商資訊卡：廠商名稱、聯絡方式、專案數量
  • 指標卡片：合約總金額、已開立、已付款、未結清金額
  • 進行中專案：目前專案清單，附帶狀態與健康度指標
  • 報價：管理廠商報價（建立、提交、刪除），並追蹤狀態（待處理、已提交、已核准、已拒絕）
  • 最新發票：最新的廠商發票及其狀態

客戶儀表板

  • 帳單摘要：合約金額、已開立、已付款、未結清金額
  • 我的專案：專案卡片網格，顯示狀態、核准狀態、任務進度條、PM/AM 負責人與到期日
  • 最新發票：最新發票清單及狀態標籤
  • 專案時間軸：進行中專案里程碑總覽

團隊成員 / 一般儀表板

  • 統計卡片：進行中專案、未完成任務、待處理工時表、本週工時、未結清發票
  • 最新專案：快速前往專案及其任務數量
  • 最新發票：最新發票清單及狀態

專案

透過側邊欄的專案進入。

標籤頁（依角色而異）

內部員工 — 4 個標籤頁

  1. 我的專案 — 指派給您的專案（作為 PM、AM 或團隊成員）
  2. 所有專案 — 系統中的所有專案，可依 PM、AM、客戶與廠商篩選
  3. 廠商申請 — 廠商提交、等待核准的專案申請
  4. 已封存專案 — 狀態為 ARCHIVED（已封存）的專案

廠商使用者 — 3 個標籤頁

  1. 目前專案 — 指派給您廠商的進行中專案
  2. 我的申請 — 您所提交的專案申請及其核准狀態
  3. 已封存專案 — 已封存的廠商專案

客戶使用者 — 1 個標籤頁

  1. 我的專案 — 與您所屬機構關聯的專案

廠商專案申請

廠商可點擊+ 新增申請提交專案申請。申請表單包含：名稱、類別、描述、開始/結束日期、預計人數、所需技能、計費方式、預估成本與預估工時。

申請可儲存為草稿或提交。提交後，PM/AM/管理員可在「廠商申請」標籤頁或其儀表板中核准或拒絕該申請。

客戶提案流程

對於已指派客戶的專案，在專案正式啟用前，必須先向客戶提出範圍與計費條款的提案並取得核准。這與上述的「廠商專案申請」不同——它管理的是客戶對成本的確認，而非廠商的接案申請。

  1. 擬定提案（PM、AM、管理員、超級管理員）— 設定專案的計費方式、預估成本、預估工時，若為按工時計費或混合計費，還須設定時薪。為按里程碑或混合計費的專案新增里程碑，並設定合約金額。
  2. 發送提案 — 在專案詳情頁點擊發送提案。客戶會收到電子郵件通知。若必須設定時薪但尚未設定，發送將被阻擋並顯示錯誤訊息。
  3. 客戶審核 — 客戶開啟專案後會看到提案面板，可以核准、拒絕（須填寫原因）或要求修訂（附帶備註）。
  4. 核准後 — 專案會自動進入 ACTIVE（進行中）狀態，計費方式、預估成本、預估工時、時薪，以及每個里程碑的合約金額、名稱與到期日皆會鎖定，無法再編輯。
  5. 發起新修訂 — PM/AM/管理員可點擊開始新修訂解鎖這些欄位並將提案版本號加一，隨後編輯並重新發送以待核准。

提案狀態：DRAFT（草稿）→ SENT（已發送）→ APPROVED（已核准）/ DECLINED（已拒絕）/ REVISION_REQUESTED（已請求修訂）。從後三種狀態中的任一種發起新修訂，提案都會回到 DRAFT 狀態，且版本號遞增。

客戶僅能檢視與處理屬於自己機構的專案提案。

提案歷史：每次發送提案時，系統都會永久記錄當時的條款與里程碑快照，以及客戶的回應與任何留言。點擊版本標籤旁的檢視歷史即可開啟時間軸，查看每一個歷史版本——發送了什麼內容、誰發送的、客戶如何回應，以及任何拒絕原因或修訂備註。任何後續修訂都不會刪除或覆蓋歷史記錄；即使提案已核准並再次修訂，所有先前版本依然可見。內部員工與客戶皆可檢視完整歷史。

變更請求

專案提案核准後，客戶可能會要求在原有範圍之外新增額外工作（例如額外的功能或交付項目）。為避免重新開啟並再次鎖定整份提案，系統使用獨立的變更請求來處理這類需求：

  1. 提出變更請求（PM、AM、管理員、超級管理員）— 在專案詳情頁點擊+ 新增變更請求，填寫標題、選填的描述、追加成本，並可選擇新增一個或多個新里程碑（名稱、到期日、金額）。
  2. 客戶審核 — 客戶會在專案頁面看到該變更請求及其他歷史請求，可以核准、拒絕（須填寫原因）或要求修訂（附帶備註）——流程與提案審核相同。
  3. 核准後 — 變更請求中列出的所有里程碑會自動建立於專案中，專案的預估成本會增加相應的追加成本。原先核准的提案及其歷史記錄不受影響。
  4. 遭拒絕或要求修訂後 — 該變更請求會永久保留，並顯示客戶的回應與備註。若需處理回饋意見，須提出新的變更請求；原請求不會被編輯或刪除。

一個專案在其生命週期內可擁有任意數量的變更請求，每一筆的完整歷史——包括遭拒絕或被取代的記錄——都會持續保留可見。

專案詳情頁

點擊任一專案名稱以開啟詳情頁，該頁面包含四個標籤頁：

  1. 工作項目 — 管理里程碑與任務。建立/編輯/刪除里程碑僅限超級管理員、管理員、AM 與 PM。每個里程碑可設定合約金額，用於在標記完成後向客戶請款（詳見上述「客戶提案流程」）；一旦專案提案獲得核准，該里程碑的名稱、到期日與合約金額皆會鎖定。建立/編輯/刪除任務（以及點擊狀態標籤快速切換任務狀態）對管理者在任意專案中皆可使用，團隊成員、承包商與廠商聯絡人則僅限於其被指派的專案。此標籤頁也可檢視團隊成員面板。
  2. 報價 — 該專案的廠商報價（檢視、建立與管理；狀態包含：PENDING 待處理、SUBMITTED 已提交、APPROVED 已核准、REJECTED 已拒絕）
  3. 預算 — 專案預算追蹤；建立與管理預算明細項目
  4. 文件 — 上傳與管理專案文件

任務

透過側邊欄的任務進入（僅限內部員工與廠商使用者）。

檢視任務

任務可以表格檢視（附帶狀態篩選標籤：全部、待辦、進行中、審核中、已完成），也可以看板檢視（看板風格欄位配置）。使用專案下拉選單依專案篩選。此檢視會顯示所有專案中的任務，無論是否指派給您——瀏覽不受限制，但編輯權限受限（詳見下文）。

建立任務

點擊+ 建立並填寫：

欄位是否必填說明
名稱是任務名稱
描述否任務詳情
狀態否TODO（待辦）、IN_PROGRESS（進行中）、REVIEW（審核中）、DONE（已完成）
優先等級否LOW（低）、MEDIUM（中）、HIGH（高）、URGENT（緊急）
專案是該任務所屬的專案
里程碑否關聯至某個專案里程碑
負責人否被指派該任務的使用者
到期日否任務期限
預估工時否工時估算
是否計費否切換該任務是否計費

「專案」與「負責人」選項範圍取決於您的權限：超級管理員/管理員/AM/PM 可檢視所有專案，並可指派給與所選專案相關的任何人。團隊成員、承包商與廠商聯絡人僅能看到指派給自己的專案，且只能指派給與該專案相關的人員。

快速切換狀態

點擊任一任務上的狀態標籤，可依序循環切換：TODO（待辦）→ IN_PROGRESS（進行中）→ REVIEW（審核中）→ DONE（已完成）（表格檢視），或在看板檢視中將卡片拖曳至不同欄位。僅當您有權管理該任務時才可使用——超出您指派專案範圍的任務將無法編輯、刪除或變更狀態。

時間追蹤

透過側邊欄的時間追蹤進入（僅限內部員工與廠商使用者）。

記錄工時

時間紀錄會顯示您所有的工時記錄。點擊+ 新增紀錄以建立一筆：

欄位是否必填說明
日期是工作日期
專案是所從事的專案
任務否具體任務

即時計時器

使用開始計時按鈕即時記錄工時。完成後點擊停止，開始/結束時間會自動帶入紀錄表單。

篩選

可依專案或日期範圍（開始與結束日期）篩選您的工時記錄。

已鎖定紀錄

已包含於已提交工時表中的工時記錄會被鎖定，無法編輯或刪除，並會顯示鎖形圖示。

7

工時表

透過側邊欄的工時表進入（僅限內部員工與廠商使用者）。

我的工時表

建立工時表，將您的工時記錄整合提交以待核准。

  1. 點擊+ 新增工時表
  2. 選擇期間開始與期間結束日期
  3. 系統會自動納入相符的工時記錄
  4. 提交工時表以待核准，或儲存為草稿

工時表狀態

  • DRAFT（草稿） — 尚未提交；可編輯或刪除
  • SUBMITTED（已提交） — 等待審核；記錄已鎖定
  • APPROVED（已核准） — 已由審核人核准
  • REJECTED（已拒絕） — 已附原因遭拒；請查看回饋後重新提交

點擊任一工時表即可展開檢視其中所有關聯的工時記錄與工時數。

審核工時表（PM、AM、管理員、超級管理員）

待審核區域顯示所有團隊成員提交的工時表。

  1. 點擊某個工時表以展開並審核其工時記錄
  2. 點擊核准以核准該工時表
  3. 點擊拒絕並填寫原因以退回

已核准工時表區域顯示先前已核准的工時表，包括總工時、可計費工時與核准人。

8

發票

透過側邊欄的發票進入（所有人可見）。

標籤頁

  • 客戶發票 — 發送給客戶的發票（內部員工與客戶可見）
  • 廠商發票 — 來自廠商的發票（內部員工與廠商可見）

使用狀態篩選器依目前狀態篩選發票。

客戶發票

建立客戶發票（PM、AM、管理員）

  1. 點擊+ 新增客戶發票
  2. 選擇專案與客戶
  3. 新增明細項目，填寫描述、數量、單價
  4. 設定稅率與幣別（自動由客戶設定帶入）
  5. 儲存為草稿

依里程碑產生（PM、AM、管理員）

對於按里程碑或混合計費的專案，點擊⚡ 依里程碑產生並選擇專案，系統會自動建立一張草稿發票，為每個已完成的里程碑產生一筆明細項目，並使用該里程碑的合約金額。已在已提交、已核准或已付款發票中向客戶請款過的里程碑不會重複計入。

發票生命週期

DRAFT（草稿）──► SENT（已發送）──► APPROVED（已核准）──► PAID（已付款）
                                              └──► OVERDUE（逾期，若超過到期日）
  • DRAFT（草稿） — 可編輯與刪除
  • SENT（已發送） — 已發送給客戶以待審核
  • APPROVED（已核准） — 客戶或 AM 已核准該發票
  • PAID（已付款） — 已收到付款
  • OVERDUE（逾期） — 已超過到期日仍未付款

廠商發票

建立廠商發票（廠商聯絡人）

  1. 點擊+ 建立發票
  2. 從已核准的報價中選擇（自動帶入詳細資訊）
  3. 新增明細項目
  4. 儲存為草稿或提交

廠商發票生命週期

DRAFT（草稿）──► SUBMITTED（已提交）──► APPROVED（已核准）──► PAID（已付款）
                              └──► REJECTED（已拒絕）
                              └──► REVISION_REQUESTED（已請求修訂）──►（建立新版本）
  • SUBMITTED（已提交） — 等待 PM/AM 審核
  • APPROVED（已核准） — PM/AM 已核准該發票
  • REJECTED（已拒絕） — AM 已拒絕該發票（附帶備註）
  • REVISION_REQUESTED（已請求修訂） — PM/AM 要求修改；系統會建立新版本供廠商編輯並重新提交

當有待處理的廠商發票時，審核佇列提示條會向 PM/AM 使用者提示數量與總金額。

發票詳情頁

點擊任一發票即可檢視完整詳情：包含狀態、日期與金額的發票標頭資訊；明細項目表格；付款歷史；以及依您的角色與發票狀態顯示的操作按鈕（提交、核准、拒絕、要求修訂）。

記錄付款

對於已核准的發票，經授權的使用者可以記錄付款：

  1. 開啟發票詳情頁
  2. 點擊記錄付款
  3. 輸入付款金額與日期
  4. 全額付款後，發票狀態會自動更新為 PAID（已付款）
9

財務

透過側邊欄的財務進入（僅限管理員、超級管理員、AM、PM）。

摘要卡片

六張卡片提供財務概覽：

  • 草稿 — 草稿發票的總金額（管道中）
  • 已開立總額 — 所有已發送的發票
  • 待審核 — 等待核准的發票
  • 已核准 — 已核准發票總額
  • 已付款 — 已收到的付款總額
  • 未結清 — 未付款餘額

專案明細表

詳細表格顯示每個專案的財務資料：專案名稱與計費方式、幣別、發票數量、草稿/已開立/待審核/已核准/已付款/未結清金額、付款進度條，以及底部合計列。

篩選

  • 廠商 — 依廠商機構篩選
  • 客戶 — 依客戶機構篩選
10

報告

透過側邊欄的報告進入（僅限管理員、超級管理員、AM、PM）。

工時報告標籤頁

  • 摘要卡片：總工時、專案數量、記錄數量
  • 依專案分布：各專案工時及佔比長條圖
  • 紀錄日誌表：詳細工時記錄（最多 50 列）
  • 篩選：專案下拉選單、日期範圍（起訖日期）

發票報告標籤頁

  • 摘要卡片：已開立總額、已收款、未結清
  • 依狀態分布：依狀態分組的發票金額（草稿、已發送、已付款、逾期）
  • 發票表格：完整清單，含已收款與餘額欄位
  • 篩選：客戶下拉選單、日期範圍（起訖日期）

管理面板

透過側邊欄的管理面板進入（僅限管理員、超級管理員）。管理面板包含三個標籤頁：

使用者標籤頁

管理系統中的所有使用者帳號。

建立使用者

  1. 點擊+ 新增使用者
  2. 填寫：姓名（必填）、電子郵件（必填）、密碼（必填）、職稱
  3. 選擇使用者類型：內部員工、廠商或客戶
  4. 若為廠商/客戶：選擇所屬機構
  5. 切換指派的角色（可指派多個角色）
  6. 設定啟用/未啟用狀態

可選角色：超級管理員、管理員、客戶經理、專案經理、團隊成員、承包商、廠商聯絡人、客戶

使用者表格顯示：姓名與職稱、電子郵件、公司類型標籤（內部員工/廠商/客戶）、角色標籤、啟用狀態。

客戶標籤頁

管理客戶機構。

欄位說明
名稱必填
聯絡電子郵件
聯絡電話
地址
付款條件預設："Net 30"（30 天內付款）
幣別下拉選擇
啟用/未啟用狀態切換

廠商標籤頁

管理廠商機構。

欄位說明
名稱必填
聯絡電子郵件
電話
地址
預設時薪顯示為「$X/小時」
幣別
啟用/未啟用狀態切換
12

個人檔案

點擊側邊欄底部的編輯個人檔案，或前往個人檔案頁面。

可編輯欄位

  • 姓名 — 您的顯示名稱
  • 電子郵件 — 您的電子郵件地址（Google OAuth 使用者無法編輯）
  • 職稱 — 您的職稱

唯讀資訊

  • 使用者類型標籤（內部員工、廠商或客戶）
  • 公司名稱（廠商/客戶使用者顯示）
  • 驗證方式標籤（本地帳號或 Google）
  • 角色 — 您所指派的角色標籤

變更密碼（僅限本地帳號）

  1. 輸入您的目前密碼
  2. 輸入您的新密碼（至少 8 個字元）
  3. 確認新密碼
  4. 點擊更新密碼

Google OAuth 使用者無法變更密碼，也不會看到此區塊。
